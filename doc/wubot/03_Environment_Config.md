## 1. 物理仿真参数

```python
self.decimation = 4          # 每 4 个物理步做 1 次策略决策
self.sim.dt = 0.005          # 物理步长 = 200 Hz
self.episode_length_s = 30.0 # 每个 episode 最长 30 秒
```

### 1.1 频率关系

```
物理引擎 (PhysX):     200 Hz  (dt = 0.005s)
RL 策略决策:           50 Hz   (200 / decimation=4)
动捕参考数据:          50 Hz   (fps=50, 与策略同步)
C++ 部署电机控制:      1000 Hz (真机伺服频率)
```

> 💡 **策略频率 = 参考数据频率 = 50Hz**，这不是巧合——NPZ 在 csv_to_npz 阶段就降采样到了 50fps，正好对应策略的决策频率。

## 2. G1 机器人物理模型

### 2.1 29 个关节分组

G1 有 29 个有效关节，按功能分为 5 组：

| 组别 | 关节 | 数量 | 电机型号 | 特点 |
|------|------|------|---------|------|
| **腰部** | waist_yaw/roll/pitch | 3 | 混合 | 躯干旋转 |
| **左腿** | hip_yaw/roll/pitch, knee, ankle_pitch/roll | 6 | 7520系列 | 大力矩 |
| **右腿** | 同上 | 6 | 7520系列 | 大力矩 |
| **左臂** | shoulder_pitch/roll/yaw, elbow, wrist_roll/pitch/yaw | 7 | 5020/4010 | 低力矩 |
| **右臂** | 同上 | 7 | 5020/4010 | 低力矩 |

### 2.2 驱动器参数来源

PD 控制器的刚度/阻尼源自**真实电机的物理测量值**：

```
固有频率: ω_n = 10 × 2π ≈ 62.8 rad/s  (电机 10Hz 固有振频)
阻尼比:   ζ = 2.0                       (过阻尼, 无振荡)
```

| 电机型号 | 用于 | 刚度 (Nm/rad) | 阻尼 (Nms/rad) | 力矩限制 (Nm) |
|---------|------|---------|--------|-----------|
| 7520-22 | hip_roll, knee | 99.1 | 6.31 | 88 |
| 7520-14 | hip_pitch/yaw | 40.2 | 2.56 | 56 |
| 5020 | ankle, shoulder | 14.3 | 0.91 | 25 |
| 4010 | wrist | 16.8 | 1.07 | 25 |

### 2.3 Action Scale 的物理含义

```python
action_scale = 0.25 × effort_limit / stiffness
```

**含义**：当网络输出 ±1 时，关节目标角度偏移 = ±(effort_limit/stiffness) × 0.25

这确保了：
- 不同电机型号有合理的运动范围
- 网络输出范围 [-1, 1] 映射到物理上合理的角度变化
- 大力矩电机（腿部）运动范围较大，小力矩电机（手腕）运动范围较小

## 3. 观测空间

### 3.1 Policy 观测（部署时使用，共 154 维）

| 观测量 | 维度 | 噪声 | 来源 | 说明 |
|--------|------|------|------|------|
| `motion_command` | 58 | 无 | NPZ当前帧 | 目标关节角度(29) + 速度(29) |
| `motion_anchor_ori_b` | 6 | ±0.05 | 仿真计算 | 锚点朝向误差(旋转矩阵前2列) |
| `base_ang_vel` | 3 | ±0.2 | IMU | 机体角速度 |
| `joint_pos_rel` | 29 | ±0.01 | 编码器 | 关节角度(相对默认值) |
| `joint_vel_rel` | 29 | ±0.5 | 编码器 | 关节速度 |
| `last_action` | 29 | 无 | 上一帧 | 上一帧的动作输出 |
| **合计** | **154** | | | |

### 3.2 Critic 特权观测（仅训练时使用）

Critic 额外获得策略看不到的"上帝视角"信息：

| 额外观测 | 说明 |
|---------|------|
| `base_lin_vel` | 根节点线速度真值 |
| `motion_anchor_pos_b` | 锚点位置误差真值 |
| `body_pos` | 14个body位置真值 |
| `body_ori` | 14个body朝向真值 |

> 💡 **为什么分两套观测？** Critic（价值网络）在训练时评估"当前状态有多好"，给它更多信息能更准确评估。Actor（策略网络）只能用部署时能拿到的传感器数据。这叫**Asymmetric Actor-Critic**。

### 3.3 为什么加噪声？

观测噪声模拟真实传感器的不精确：

| 噪声类型 | 真实对应 |
|---------|---------|
| joint_pos ±0.01 rad | 编码器精度 (~0.6°) |
| joint_vel ±0.5 rad/s | 速度估计误差 |
| base_ang_vel ±0.2 rad/s | IMU 漂移 |
| anchor_ori ±0.05 | 状态估计误差 |

训练时加噪声 → 策略学会应对不精确的输入 → 部署到真机时更鲁棒。

## 4. 域随机化 (Domain Randomization)

在训练启动和运行过程中随机化物理参数，让策略不过拟合到完美仿真：

| 参数 | 范围 | 模式 | 目的 |
|------|------|------|------|
| 摩擦系数 | 静摩擦[0.3,1.6], 动摩擦[0.3,1.2] | startup | 适应不同地面 |
| 关节默认位置 | ±0.01 rad | startup | 模拟校准误差 |
| 躯干质心偏移 | x±0.025, y±0.05, z±0.05 m | startup | 模拟负载变化 |
| 外力推扰 | 线速度±0.5 m/s, 角速度±0.78 rad/s | interval 2~5s | 鲁棒性训练 |

### 武术 vs 舞蹈的域随机化差异

| 参数 | 武术 | 舞蹈 | 原因 |
|------|------|------|------|
| `push_robot` 间隔 | **2~5秒** | 1~3秒 | 武术动作复杂，推扰太频繁会阻碍学习 |
| 其他 | 相同 | 相同 | 物理环境随机化需求一致 |

## 5. 终止条件

当以下任一条件触发时，episode 立即结束，机器人重置：

| 条件 | 阈值 | 检查内容 | 设计理由 |
|------|------|---------|---------|
| `time_out` | 30秒 | episode 时长 | 防止无限运行 |
| `anchor_pos` | 0.25m | 锚点(躯干)Z轴偏离 | 防止蹲下或跳起太多 |
| `anchor_ori` | 0.8 | 躯干朝向偏差 | 防止翻倒 |
| `ee_body_pos` | **0.6m** | 末端执行器Z轴偏离 | 允许踢腿抬脚 |

### 5.1 ee_body_pos 阈值的故事 (v4 关键修改)

```
v1-v3: threshold = 0.25m
问题: 正踢腿时脚部升高 ~0.8m，0.25m 阈值在踢腿中途直接终止 episode！
      策略学到："踢腿 = 被惩罚" → 永远不踢腿

v4:   threshold = 0.6m
效果: 允许脚部升高到 0.6m（覆盖大部分踢腿动作）
      策略终于能学到完整的踢腿动作
```

> ⚠️ **对于 Side Kick / Roundhouse Kick**，腿侧面/后方运动更极端，可能需要进一步放宽到 0.8m。
