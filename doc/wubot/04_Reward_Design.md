## 1. 奖励架构总览

```
Total Reward = Σ(weight_i × reward_i)

正则化惩罚 (负权重 → 防止过激动作):
  ├── joint_acc_l2         × -2.5e-7    关节加速度
  ├── joint_torques_l2     × -1e-5      关节力矩
  ├── action_rate_l2       × -0.05      动作变化率 (v5: 从-0.1减半)
  ├── joint_pos_limits     × -10.0      关节极限
  └── undesired_contacts   × -0.1       非预期接触

锚点追踪 (躯干):
  ├── anchor_pos           × 0.5 σ=0.3  躯干位置
  └── anchor_ori           × 1.0 σ=0.5  躯干朝向 (v5: w↑, σ↓)

全身追踪 (14个body):
  ├── body_pos             × 1.5 σ=0.5  body位置
  ├── body_ori             × 1.5 σ=0.8  body朝向
  └── joint_pos            × 3.0 σ=0.6  关节角度 (v5: w↑3.0, σ↓0.6)

末端执行器 (4个手脚, v5新增):
  ├── ee_pos               × 3.0 σ=0.2  手脚位置 (极严格)
  └── ee_lin_vel           × 1.0 σ=1.0  手脚速度 (打击利落度)

速度追踪 (辅助):
  ├── body_lin_vel         × 0.5 σ=1.5  线速度
  └── body_ang_vel         × 0.5 σ=4.0  角速度
```

## 2. exp 核函数——理解一切的基础

**所有追踪 reward 都使用相同的公式**：

$$R = \exp\left(-\frac{\text{mean}(\|q_{\text{ref}} - q_{\text{robot}}\|^2)}{\sigma^2}\right)$$

### 2.1 直觉理解

- 完美匹配（error=0）→ R = 1.0
- error 越大 → R 指数衰减趋近 0
- **σ 控制"宽容度"**：σ 越小 → 衰减越快 → 要求越严格

### 2.2 σ 值选择指南

| σ值 | error=0.3 | error=0.5 | error=0.8 | 适用场景 |
|-----|-----------|-----------|-----------|---------|
| **0.2** | 0.11 | 0.00 | 0.00 | 极严格（末端执行器位置） |
| **0.3** | 0.37 | 0.06 | 0.00 | 严格（锚点位置） |
| **0.5** | 0.70 | 0.37 | 0.08 | 中等（锚点朝向、body位置） |
| **0.6** | 0.78 | 0.51 | 0.17 | 中等偏松（关节角度） |
| **0.8** | 0.87 | 0.67 | 0.37 | 宽松（body朝向） |
| **1.0** | 0.91 | 0.78 | 0.53 | 很宽松（末端速度） |
| **1.5** | 0.96 | 0.89 | 0.74 | 极宽松（body线速度） |
| **4.0** | 0.99 | 0.98 | 0.96 | 几乎不区分（body角速度） |

### 2.3 σ 的"甜蜜区间"

```
σ 太小 (< 初始误差):
  → 初始 reward ≈ 0 → 梯度消失 → 策略学不到任何东西
  → 例: body_ori 初始误差 ~0.79 rad, σ=0.4 时 reward=0.02 (死！)

σ 太大 (>> 初始误差):
  → 初始 reward ≈ 1 → 策略觉得"已经很好了" → 缺乏改进动力
  → 例: body_ang_vel σ=4.0, 几乎不区分好坏

σ 刚好 (≈ 初始误差):
  → 初始 reward 在 0.3~0.7 之间 → 有梯度信号 → 策略能持续改进
  → 例: body_pos σ=0.5, 初始误差 ~0.24m → reward=0.77 (有空间提升)
```

> 💡 **调参核心规则**：先估算初始误差，然后让 σ ≈ 初始误差，使初始 reward 落在 0.3~0.7 之间。

## 3. 每个 Reward 项详解

### 3.1 正则化惩罚（负权重）

这些项防止策略学到"有效但难看"的动作：

| 项 | 权重 | 作用 | 太大会怎样 | 太小会怎样 |
|----|------|------|-----------|-----------|
| `joint_acc_l2` | -2.5e-7 | 平滑关节加速度 | 动作迟钝 | 关节抖动 |
| `joint_torques_l2` | -1e-5 | 减少力矩 | 动作无力 | 电机过载 |
| `action_rate_l2` | **-0.05** | 平滑动作变化 | **无法爆发**🥋 | 动作抖动 |
| `joint_pos_limits` | -10.0 | 远离关节极限 | 运动范围受限 | 可能损坏关节 |
| `undesired_contacts` | -0.1 | 避免非法接触 | 过于保守 | 摔倒不被惩罚 |

> ⚠️ **`action_rate_l2` 是武术调参的关键！** v4 用 -0.1 太大，压制了出拳/踢腿的爆发力。v5 降到 -0.05，释放瞬间爆发。

### 3.2 锚点追踪（躯干稳定）

| 项 | 权重 | σ | 追踪目标 |
|----|------|---|---------|
| `anchor_pos` | 0.5 | 0.3 | 躯干在空间中的位置（Z轴为主） |
| `anchor_ori` | **1.0** | **0.5** | 躯干的朝向（防佝偻）|

> **v5 变化**：anchor_ori 从 w=0.5/σ=0.8 → w=1.0/σ=0.5。原因：踢腿时策略为保持平衡会大幅弯腰（"佝偻"），收紧躯干朝向约束强迫保持直立。

### 3.3 全身追踪

| 项 | 权重 | σ | 追踪目标 | 初始误差参考 |
|----|------|---|---------|-------------|
| `body_pos` | 1.5 | 0.5 | 14个body的CoM位置 | ~0.24m |
| `body_ori` | 1.5 | 0.8 | 14个body的朝向 | ~0.79rad |
| `joint_pos` | **3.0** | **0.6** | 29个关节角度 | 取决于初始化 |

### 3.4 为什么需要 joint_pos？(v4 核心修复)

**问题**：`body_pos` 追踪 link 的质心，`body_ori` 追踪朝向。但机器人可以用**完全错误的关节配置**达到近似正确的质心轨迹！

```
正确的膝盖弯曲:        错误但质心相同:
     O                      O
    /|\                    /|\
    / \     ← 质心 →      /  \
   /   \    相同位置      /    |
  足   足                足    足
  (膝弯90°)             (膝弯45° + 踝过弯)
```

**解决方案**：`joint_pos` reward 直接比较 29 个关节角度 → 消除"对的位置、错的姿势"。

### 3.5 末端执行器专项追踪 (v5 新增)

| 项 | 权重 | σ | 追踪目标 |
|----|------|---|---------|
| `ee_pos` | **3.0** | **0.2** | 手脚的精确位置 |
| `ee_lin_vel` | 1.0 | 1.0 | 手脚的打击速度 |

**设计意图**：σ=0.2 对于手脚位置极其苛刻——0.3m 误差时 reward 仅 0.11。这确保"拳头打到位、脚踢到高度"——春晚级精准。

### 3.6 速度追踪（辅助角色）

| 项 | 权重 | σ | 说明 |
|----|------|---|------|
| `body_lin_vel` | 0.5 | 1.5 | 线速度追踪，权重低 |
| `body_ang_vel` | 0.5 | 4.0 | 角速度追踪，极宽松 |

> 💡 速度追踪权重故意压低（0.5 vs 位置的 1.5），σ 故意放宽。原因：速度追踪会和位置追踪"打架"——位置对了但速度方向不对，就会产生冲突信号。让位置主导，速度辅助。

## 4. 武术 vs 舞蹈的奖励差异

| 奖励项 | 武术 (w/σ) | 舞蹈 (w/σ) | 武术修改原因 |
|--------|----------|----------|------------|
| `body_pos` | 1.5 / **0.5** | 1.0 / 0.3 | 初始误差 0.24m, σ=0.3 梯度消失 |
| `body_ori` | **1.5** / **0.8** | 1.0 / 0.4 | 初始误差 0.79rad, σ=0.4 时 R=0.02 |
| `joint_pos` | **3.0** / 0.6 | ❌ 不存在 | v4 核心修复，防止错误姿势 |
| `ee_pos` | **3.0** / **0.2** | ❌ 不存在 | v5 新增，手脚精准打击 |
| `ee_lin_vel` | 1.0 / 1.0 | ❌ 不存在 | v5 新增，打击利落度 |
| `body_lin_vel` | **0.5** / **1.5** | 1.0 / 1.0 | 降权+放宽，不抢占位置 |
| `body_ang_vel` | **0.5** / **4.0** | 1.0 / 3.14 | 同上 |
| `anchor_ori` | **1.0** / **0.5** | 0.5 / 0.4 | 防佝偻 |
| `action_rate` | **-0.05** | -0.1 | 解锁爆发力 |
| ee终止阈值 | **0.6m** | 0.25m | 允许踢腿 |

## 5. v1 → v5 演化历程

| 版本 | 核心问题 | 修复 |
|------|---------|------|
| **v1-v2** | body_ori σ=0.4 → 梯度消失，策略完全学不到 | — |
| **v3** | 放宽 σ (ori 0.4→0.8, pos 0.3→0.5) | ✅ 策略开始学习，但姿势怪异 |
| **v4** | "对的位置、错的姿势"——没有关节角度约束 | ✅ 新增 `joint_pos` reward |
| **v4** | 踢腿被终止——ee 阈值 0.25m 太严 | ✅ 放宽到 0.6m |
| **v5** | 手臂乱挥——joint_pos w=2.0 不够 | ✅ w 2.0→3.0, σ 0.8→0.6 |
| **v5** | 躯干佝偻——anchor_ori 太松 | ✅ w 0.5→1.0, σ 0.8→0.5 |
| **v5** | 动作不爆发——action_rate -0.1 压制 | ✅ 降到 -0.05 |
| **v5** | 手脚打击不精准 | ✅ 新增 ee_pos/ee_vel |

> 💡 **每一次修改都有因果关系**。理解了"为什么改"比记住"改成多少"更重要。

## 6. 自适应采样 (Adaptive Sampling)

训练时不是均匀采样动作片段，而是聚焦困难部分：

```
时间轴: |---起手式---|---准备---|-!-踢腿-!-|---收腿---|---结束---|
失败率:      低          低        高!         中          低
采样概率:    5%         10%       50%!        25%         10%
```

关键参数：
- `adaptive_alpha=0.002`: EMA 平滑系数（慢更新失败率统计）
- `adaptive_kernel_size=3`: 卷积核大小（关注相邻 bin）
- `adaptive_lambda=0.8`: 核权重衰减
- `adaptive_uniform_ratio=0.1`: 10% 均匀采样（防止遗忘简单片段）
